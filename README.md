

# 工作量及报价

## 阶段工作量分布
> 整个项目大的工作阶段工作量权重遵照下列比例，权重偏差控制在10%之内

序号|工作阶段|工作量权重
----|--------|----------
1|需求分析|7%
2|系统设计|25%
3|程序开发|50%
4|系统测试|15%
5|系统部署|3%

## 角色工作量权重及费用

序号|角色|工作量权重|费用（元/人天）|工作职责描述
----|----|----------|----------------|------------
1|项目经理|18%|1050|1、负责编制总体项目计划；参与评审配置计划、质量保证计划和测试计划；<br>2、制定项目管理制度，进行员工绩效考核和奖惩。<br>3、负责控制项目的总体进度、成本、质量和风险；<br>4、负责与客户相关部门(决策委员会、系统分析小组)联络沟通和任务协调；<br>5、负责公司内部资源对项目组的调配/协调；<br>6、负责定期组织项目阶段性评审；<br>7、定期编制项目总结提交给双方的管理部门。
2|架构设计师|20%|1000|1、负责系统的架构设计和变更；<br>2、提交设计书，提交业务分析组审核；<br>3、建议系统各部分的关键技术难点和所需时间，提交业务分析组审核；<br>4、参与项目经理制定开发项目计划过程；<br>5、评审架构设计和设计工程师的设计文档；<br>6、填写相关的过程统计文档。<br>7、项目经理分配的其它工作；
3|需求分析员|21%|1000|1、分析用户的需求，协商完成软件需求规格说明书；<br>2、参与软件配置控制委员会活动，管理需求变更；<br>3、评审系统架构和测试文档；
4|程序员|9%|800|1、按照设计工程师提交的设计文档，进行编码。<br>2、对完成的编码部分进行单元测试(提交必要的测试用例)。<br>3、编制工作文档（工程文档、开发文档、测试文档）；<br>4、填写相关的过程统计文档。<br>5、评审软件开发工程师、开发研讨。<br>6、完成开发经理分配的其它开发和工程任务；
5|软件测试工程师|2%|800|1、编制测试计划、测试方案和测试用例；<br>2、进行集成测试和系统测试和确认测试；<br>3、执行测试计划；<br>4、编写测试报告<br>5、填写相关的过程统计文档。
6|网页设计师|11%|800|1、实现系统分析师提出的用户界面要求。<br>2、完成前端展现的页面框架设计。<br>3、完成向导、帮助的界面设计。<br>4、完成页面、按钮设计。<br>5、编写必要的脚本程序。<br>6、填写相关的过程统计文档。<br>7、项目经理分配的其它工作。
7|文档编辑|13%|700|1、完成项目的周、月例会等会议纪要等文档的编辑<br>2、检查项目文档是否增加到项目配置库中，是否增加了版本控制。<br>3、完成项目的帮助手册、等文档的编辑<br>4、项目经理分配的其它工作；
8|部署工程师|6%|550|1、制定编写实施方案、实施计划、应急计划等。<br>2、完成项目实施所有软、硬件和网络等环境的准备。<br>3、根据配置管理员提交的项目版本进行项目实施。<br>4、项目经理分配的其它工作。
-|均价|-|898|-

- 备注

1. 工作量的预估采用专家意见法预估，专家数量不得少于2人；
2. 人力成本估算以公司上年度平均薪酬W（含社会保险、各种补贴）作为基数；
3. 估算结果的计算公式：（最小工作量+4×最可能工作量+最大工作量）/6；
4. 核定工作量是指项目全过程的工作量；
5. 本表格是项目立项评审的组成部分，存档备查。
